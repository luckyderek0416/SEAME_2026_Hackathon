# 미구현 / 미완성 기능 정리 (트랙 미션 기준)

기준: OT 자료 10~19p (트랙·미션 소개) vs 현재 코드 (2026-07-03).
레이스: 7/15 (1차), 7/16 (2차). **굵은 항목이 레이스 전 필수.**

## 미션별 구현 현황 요약

| 트랙 구간 | 규정 요약 | 구현 상태 |
|---|---|---|
| 출발 (신호등) | 초록불 점등 → 출발. 미인식 = 미션 실패 | ✅ 구현 (`WAIT_GREEN`, green 3프레임 연속) |
| S자 주행 | 미션 없음, 저속 권장. 이탈 +30s | ✅ 구현 (차선 PID + 곡률 비례 감속) |
| 좌/우 갈림길 | 랜덤 표지판 방향대로 진입. 실패 시 재출발 | ⚠️ 부분 구현 (아래 1번) |
| 동적 장애물 (아루코) | 등장 시 정지, 퇴거 시 출발. 미리 멈추거나 일찍 출발 = 실패 | ✅ 구현 (`OBSTACLE_STOP`) — 임계 튜닝 필요 |
| 도착 (신호등) | 도착 후 빨간불 점등 시 정지. 미인식 +30s | ✅ 구현 (`FINISH`→`DONE`) |
| 회전교차로 (In 코스, 옵션) | 1회 이상 회전 후 탈출. 미달 = 미션 실패 | ⚠️ 구현됐으나 캘리브레이션 필수 (아래 2번) |
| 역방향 트랙 배정 | 당일 추첨으로 정/역방향 배정 | ✅ `race_dir` 하나로 flip — 실주행 미검증 (아래 3번) |

## 1. 갈림길: 분기 "구간" 인식이 없음 (시간 기반 편향만)

**현재 동작:** YOLO가 left/right 표지판을 보면 latch → `fork_bias`(0.4)만큼 조향
편향을 더함. 갈림길이 "어디부터 어디까지"인지는 모르고, 표지판 마지막 인식 후
`fork_hold_s`(기본 6초, 오늘 추가) 지나면 편향 해제.

**미구현:**
- 갈림길 입구(점선 개방, `junction` 신호)와 연동한 정확한 분기 시점 판단.
  현재는 표지판을 본 순간부터 편향이 걸려, 갈림길 도달 **전** 직선 구간에서
  차선 추종을 한쪽으로 당길 수 있다.
- 분기 후 "올바른 쪽으로 들어갔는지" 자가 확인 로직.

**레이스 전 할 일:** 트랙에서 `fork_bias`(세기)와 `fork_hold_s`(유지 시간)를
실측 튜닝. 표지판→갈림길 도달 소요 시간을 재서 `fork_hold_s`를 거기에 맞춰라.

## 2. 회전교차로 랩 카운트: 캘리브레이션 안 된 추정값 2개 ★필수

랩 판정은 3-표결(junction 재등장 / 조향 적분 yaw proxy / 시간)인데, 이 중 둘이
**임의 초기값**이다:
- `yaw_lap_threshold: 6.0` — 실제 한 바퀴 조향 적분값 실측 필요
- `nominal_loop_time_s: 8.0` — 레이스 속도로 한 바퀴 실측 필요

**미보정 상태로 In 코스를 뛰면 조기/지연 탈출 위험.** 트랙에서 한 바퀴 돌려
debug 로그로 두 값을 읽고 기본값에 반영해야 한다. (In 코스 포기하고 Out 코스만
뛴다면 이 항목은 무시 가능 — 단, 회전교차로는 미션 포기가 불가하므로 In 코스
선택 시 반드시 보정.)

## 3. 역방향(시계방향) 트랙 실주행 미검증

`race_dir:=right` 하나로 회전교차로 방향·junction 탐색 방향이 함께 뒤집히게
만들어져 있지만 **실제로 역방향 주행 테스트를 한 적이 없다.** 당일 역방향
트랙을 배정받을 확률이 50%이므로, 연습 시간에 반드시 한 번 검증할 것.

## 4. BEV(버드아이뷰) 소스 좌표 미캘리브레이션 ★필수

`use_birdeye` 기본 ON인데 `birdeye_src_ratio`는 실제 트랙/카메라 각도에 맞춘
값이 아니라 일반 추정값. **미보정 BEV는 오히려 인식을 해칠 수 있다.**
- 트랙에서 모니터 HSV Lane 패널의 SRC 뷰(노란 사각형)를 보며 조정, 또는
- 시간 없으면 `ros2 param set /lane_node use_birdeye false` 로 끄고 주행.

## 5. YOLO 실전 검증 (모델은 있음)

NCNN 변환 모델(7/1 학습, 4클래스)이 로드되도록 배선돼 있으나, **실제 트랙
거리/조명에서 4객체 인식률을 검증한 적 없다.** 특히:
- 출발 초록불: 미인식 = 미션 실패 (가장 치명적)
- 도착 빨간불: 미인식 = +30s
- 표지판: 인식 거리가 짧으면 `fork_hold_s` 재조정 필요

`conf_threshold`(0.5)와 `infer_hz`(10)도 현장 조정 대상.

## 6. 아루코 사전 미확인

`DICT_4X4_50` + inverted 는 마커 사진에서 추정한 값. 당일 실물 마커로
`tools/identify_aruco.py` 돌려 확정할 것. 또한 `marker_area_trigger`(0.02)가
너무 민감하면 규정상 "미리 멈춤 = 미션 실패"이므로 등장 거리 기준으로 튜닝.

## 7. 차선 이탈 복구 로직 없음

lane lost 시 "offset 0으로 직진하며 재획득 대기"가 전부다. 완전히 이탈해
차선이 화면 밖이면 스스로 복귀 못 함 (규정상 팀장이 수동 복귀, +30s).
적극적 탐색(조향 스윕 등)은 미구현 — 리스크 수용하고 이탈 자체를 막는
튜닝(kp/kd, steer_slow, max_steering_delta)에 시간 쓰는 게 낫다.

## 8. 하드웨어 리스크 (소프트웨어로 못 막음)

- **I2C 버스 wedge**: 주행 진동/커넥터 문제로 재발 가능. 소프트웨어는 재시도
  +respawn까지만. **진짜 비상정지는 ESC 전원 스위치.**
- steer_center 0.2 / kp 0.6은 벤치 값 — 트랙에서 조향 반응 보고 라이브 튜닝.
